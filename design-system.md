# LandX Design System

## 1. Purpose

This document is the visual source of truth for LandX.

Its purpose is to keep:

- the marketing website consistent
- the dashboard consistent
- future pages aligned with one identity
- design decisions stable across iterations

No new page or major UI update should ignore this document.

## 2. Brand Direction

LandX should feel:

- operational
- premium
- grounded
- Arabic-first
- business-focused
- calm, not flashy
- modern, but not generic startup blue

The visual identity is based on an earthy palette:

- brown
- beige
- soft sand
- warm orange
- muted burgundy

This gives the product a stronger and more memorable market position than default SaaS blue/purple styling.

Approved design direction:

- earthy, premium, low-glare surfaces
- no bright white dashboard surfaces
- no blue glows or blue-tinted cards
- readable warm contrast across both marketing and internal product areas

## 3. Product Surfaces

LandX has two visual surfaces:

### 3.1 Marketing Website

Purpose:

- explain the product
- build trust
- present services and packages
- collect leads
- move visitors toward demo or subscription

Style:

- cinematic
- rich background lighting
- stronger gradients
- larger hero moments
- more visual storytelling

### 3.2 Dashboard App

Purpose:

- daily operational use
- fast scanning
- low-friction execution
- clarity over spectacle

Style:

- cleaner
- more restrained
- darker earthy workflow surfaces
- consistent spacing
- high readability

Both surfaces must share the same identity, but not the same intensity.

### 4.2 Usage Rules

- dark earthy backgrounds for the public website
- dark earthy surfaces for dashboard cards and forms
- orange-brown for primary emphasis
- sand/beige for support highlights
- burgundy only as secondary depth or alert accent
- internal pages should feel warm, grounded, and low-glare
- strong readability is mandatory on all dark surfaces

### 4.3 Forbidden Direction

Avoid introducing:

- default SaaS blue as a primary identity color
- purple gradients
- cold neon palettes
- bright white cards or cream-heavy app backgrounds
- pure black and pure white high-contrast surfaces unless necessary

Blue may only exist if it is explicitly re-approved later for data visualization or a separate secondary system. It is not part of the current identity.

## 5. Typography

### 5.1 Approved Fonts

- headings: `Cairo` 
- body/UI text: `Tajawal` 

Fallback:

- `"Segoe UI", Tahoma, Geneva, Verdana, sans-serif` 

### 5.2 Typography Principles

- headings should feel strong, clean, and controlled
- body text should be highly readable in Arabic
- line-height should be generous
- large hero text should never feel oversized just to fill space

### 5.3 Hierarchy Rules

- hero heading: bold, premium, controlled width
- section headings: large but relaxed
- card headings: compact and readable
- body text: airy with comfortable spacing
- muted text: soft beige-brown, not gray-blue

## 6. Spacing System

LandX should feel breathable.

Default direction:

- more whitespace between sections
- more padding inside cards
- more distance between heading and paragraph
- avoid compressed layouts

Guidelines:

- section-to-section spacing should be visually obvious
- cards should not feel crowded
- text should not sit close to borders
- grids should have enough gap to distinguish items clearly

If a section feels "dense", add space before adding decoration.

## 7. Section Structure

Every major section on the public website should follow this rhythm:

1. section divider
2. section kicker
3. heading
4. supporting paragraph
5. content grid or layout

### Section Divider Rule

Sections should be separated with:

- more vertical spacing
- a thin center-weighted divider line
- faded edges

This divider is part of the identity and should remain consistent.

## 8. Buttons

### 8.1 Primary Buttons

Use:

- warm brown/orange gradient
- earthy glow
- soft premium shadow

Never use:

- blue glow
- cold glass effect with blue tint

### 8.2 Secondary Buttons

Use:

- translucent earthy surface
- warm border
- subtle hover

### 8.3 Interaction Feel

Buttons should feel:

- refined
- responsive
- premium
- not aggressive

Hover should lift slightly, not jump dramatically.

## 9. Cards

### 9.1 Marketing Cards

Use:

- dark translucent surfaces
- warm borders
- rich blur/glass effect
- larger padding
- clear internal spacing

### 9.2 Dashboard Cards

Use:

- dark earthy surfaces
- warm contrast
- clean shadows with brown depth
- high legibility
- operational clarity

Do not use:

- white dashboard cards
- pale cream panels as the default app surface
- low-contrast brown text on pale backgrounds

### 9.3 Card Rules

- rounded corners are part of the identity
- card content should breathe
- icons should not dominate copy
- never stack text too close to the card edge

## 10. Backgrounds and Lighting

LandX backgrounds should feel atmospheric, not flat.

Approved treatments:

- radial ambient glows
- earthy gradients
- subtle animated light drift
- faint grid texture when appropriate

Rules:

- motion should be slow and soft
- background should support content, not compete with it
- glows must remain within the approved color family

## 11. Icons

Icons should be:

- clean
- modern
- minimal
- visually secondary to content

Icon containers should use warm earthy lighting, not blue-glass styling.

## 12. Header and Footer

### Header

Must feel:

- premium
- stable
- product-led

Rules:

- strong brand mark
- simple nav
- clear primary CTA
- no clutter

### Footer

Must feel:

- grounded
- complete
- structured

Rules:

- company/product summary
- key navigation
- clear action path

## 13. Dashboard Rules

The dashboard must inherit the same brand, but with lower visual drama.

Rules:

- keep workflows readable
- use the earthy palette in restrained ways
- preserve clean tables and forms
- avoid oversized marketing-style typography in operational screens
- prioritize scanning speed over visual experimentation

## 14. Forms and Tables

### Forms

- inputs must have enough padding
- labels must be clear and close to inputs
- focus state must use warm brand styling
- form groups should not look compressed

### Tables

- comfortable cell padding
- clean row separation
- no overly dark heavy borders
- support fast operational reading

## 15. Motion Principles

Motion is allowed, but it must be intentional.

Approved motion:

- slow glow drift
- reveal-up on section entry
- subtle hover lift
- soft pulse in hero lighting

Avoid:

- bouncy motion
- noisy looping animations
- excessive parallax
- flashy microinteractions

## 16. RTL and Arabic Rules

LandX is Arabic-first.

Rules:

- all layouts must work naturally in RTL
- Arabic text must be the primary design assumption
- spacing must be tested visually in Arabic, not just translated from English layouts
- typography must respect Arabic line-length and density

## 17. Consistency Rules

Before introducing any new component or page, check:

- does it use the approved palette?
- does it use the approved fonts?
- does it follow spacing rules?
- does it match the correct surface style: marketing or dashboard?
- does it preserve the brand mood?

If the answer is no, revise before implementation.

## 18. Do / Don't

### Do

- use earthy warmth
- keep sections breathable
- use premium but restrained motion
- maintain strong Arabic readability
- use consistent button and card patterns

### Don't

- reintroduce blue glows
- make hero text oversized without control
- compress cards and sections
- mix unrelated visual styles
- design a page in isolation from the system

## 19. Enforcement Rule

This file is the official visual reference for LandX.

Future work should follow it when building:

- landing pages
- internal pages
- auth screens
- settings pages
- dashboards
- forms
- pricing pages
- announcement pages

If a new design direction is needed later, this document must be updated first, then the implementation should follow it.

## 20. Dark Mode Implementation

LandX uses a dark mode design system as the default and only theme.

### 20.1 Theme File

The dark mode theme is defined in `src/themes/dark.js` and applied globally.

### 20.2 No Light Mode

This design system does not include a light mode. All surfaces are dark earthy colors as defined in the color palette.

### 20.3 Theme Application

The dark mode theme is applied at the root level and should not be overridden by individual components unless explicitly documented in this design system.

### 20.4 Consistency

All pages and components must use the dark mode colors from this design system. No component should introduce light mode colors or high-contrast white backgrounds.
